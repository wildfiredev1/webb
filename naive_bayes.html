NLP with Naive Bayes Classifier¶

In this blog post we look at text classification using the Naive Bayes classifier. We'll start with exploring and preprocessing the dataset, followed by implementation of a naive bayes classifier from scratch. We'll also look at an implementation of Gaussian Naive Bayes and compare it with scikit-learn's implementation. We discuss the effects of smoothing and also touch upon multinomial classifiers towards the end.

Preprocessing the dataset¶

Natural Language data cannot be used directly while training a machine learning model. ML models understand the language of vectors and numbers; english, not so much. While preprocessing the data, we will perform several operations such as removing stopwords, tokenizing the data etc. in an attempt to minimize the number of features that the model trains on. Also, this way, the noise is minimized in the train set.

Stopwords¶

Stop words are a set of commonly used words in any language. Usually, removing these words doesn't have any negative impacts on our model, instead it serves to reduce the noise present in the dataset. Upon removing stopwords, the remaining words are those that have more impact on the model's classfication. To learn more about stopwords, please refer: https://kavita-ganesan.com/what-are-stop-words/.

Following are the stopwords that we will remove from our dataset. This set of stop words is obtained from the "natural language toolkit (nltk)" library.

def preprocess(sent):
    sent = sent.lower() # make text lowercase
    sent = " ".join([word for word in str(sent).split() if word not in stops]) # removing stopwords
    sent = re.sub(r'[^A-Za-z0-9 ]+', '', sent) # removing non-alphanumeric characters and whitespaces (except " ")
    return sent

Stemmization¶

The next step in preprocessing is to perform stemmization. The motivation behind stemmization is as follows: Sentences in english (and all other languages) sometimes use different forms of the same word, such as fly, flies, flying etc. There may also be some words that are derived from a base word and convey nearly the same meaning. It is useful for normalization and dimensionality reduction purposes to convert all these words to their base forms. This is what stemmization does for us. It reduces the inflectional forms and sometimes derivationally related forms of a word to a common base form.[3] For instance: car, cars, car's, cars' all become car.

However, note that stemmization may also result in loss of information sometimes. Stemmization and lemmatization reduce words to what these methods consider to be their base words. They may be wrong and reduce completely unrelated words to the same base word. Worse even, they may reduce words with opposite meanings to the same base word. "Normalization and stemming consolidate words into groups that improve the “recall” for search engines but reduce precision."[4]

Tokenization¶

Finally, once we are satisfied with the words we are leaving in the dataset, we can perform tokenization. Tokenization breaks down large bodies of texts into smaller more managable chunks. Tokens are the building block of a natural language. The tokens can be either words, subwords or characters. In our case, we will be focussing on word tokenization. Word Tokenization is performed on the entire corpus (text dataset) to obtain tokens. The tokens obtained are subsequently used to prepare a vocabulary. A vocabulary is the set of unique tokens in the corpus. We can construct the vocabulary by taking all the words in the text corpus or using only the k most frequently occuring words.[4]

Simple Naive Bayes¶

The Naive Bayes Classifiers (NBC) relatively simple but effective machine learning algorithms. They are based on the concept of conditional probability and Bayes's Theorem. There are several categories of Naive Bayes classifiers such as Gaussian NB, Multinomial NB, Disccrete NB, Categorical NB etc.

When the outcome of an event is based on the occurence of one or more events, conditional probability comes into the picture. here, we try to calculate the probability of the second event (say event B) given that the first event (say event A) has already happened. If the probability of the event changes when we consider the probability of the first event, we can say that the probability of event B is dependent of the occurrence of event A. This is denoted by P(B|A).

Simple Naive Bayes employs Bayes Theorem : P(A|B) = P(B|A)* P(A)/P(B) to determine the maximum probability of a feature vector belonging to some class.

def naivebayescls(x_t):
    
    probs = np.empty(shape=num_classes)
    alpha = 10
    for y in range(num_classes):
        prob = samples_per_class[y]/num_samples # p(y)

        X = df_bow[df_bow.label == y].drop(columns='label').to_numpy()
        xt1 = x_t.copy()
        for x_f in X:
            x_t += x_f
        x_t -= xt1
        x_t += alpha
        x_t /= float(num_samples + alpha*num_features)
        probs[y] = np.prod(x_t)

    return np.argmax(probs)

Gaussian Naive Bayes¶

We'll implement a Gaussian Naive Bayes classifier from scratch. The Gaussian Naive Bayes makes use of the following equation -
$ P(x_i \mid y) = \frac{1}{\sqrt{2\pi\sigma^2_y}} \exp\left(-\frac{(x_i - \mu_y)^2}{2\sigma^2_y}\right) $

The parameters variance and mean are estimated using maximum likelihood. Following is an implementation of Gaussian Naive Bayes from scratch.

In [33]:
# Ref : https://medium.com/swlh/naive-bayes-from-scratch-c0c93ed4b826
class GaussianNaiveBayes:

    def fit(self, X, y):
        n_samples, n_features = X.shape
        self.classes = np.unique(y)
        n_classes = len(self.classes)
      
        # initialize mean, var, and prior
        self._mean = np.zeros((n_classes, n_features), dtype=np.float64)
        self._var = np.zeros((n_classes, n_features), dtype=np.float64)
        self._priors =  np.zeros(n_classes, dtype=np.float64)

        #calculate for each class
        for idx, c in enumerate(self.classes):
            X_c = X[y==c] #examples where label == c
            self._mean[idx, :] = X_c.mean(axis=0)
            self._var[idx, :] = X_c.var(axis=0)
            self._priors[idx] = X_c.shape[0] / float(n_samples) #P['the'] = num of documents containing ‘the’ / num of all documents

    def predict(self, X):
        y_pred = [self._predict(x) for x in X]
        return np.array(y_pred)

    def _predict(self, x):
        posteriors = []

        # calculate posterior probability for each class
        for idx, c in enumerate(self.classes):
            prior = np.log(self._priors[idx])
            posterior = np.sum(np.log(self._pdf(idx, x)))
            posterior = prior + posterior
            posteriors.append(posterior)
            
        # return class with highest posterior probability
        return self._classes[np.argmax(posteriors)]
            

    def _pdf(self, class_idx, x):
        mean = self._mean[class_idx]

Gaussian Naive Bayes with epsilon smoothing¶

class NaiveBayes:

    def fit(self, X, y):
        n_samples, n_features = X.shape
        self._classes = np.unique(y)
        n_classes = len(self._classes)
        self.eps = 1e-8 * np.var(X, axis=0).max() # defining the epsilon
        # calculate mean, var, and prior for each class
        self._mean = np.zeros((n_classes, n_features), dtype=np.float64)
        self._var = np.zeros((n_classes, n_features), dtype=np.float64)
        self._priors =  np.zeros(n_classes, dtype=np.float64)

        for idx, c in enumerate(self._classes):
            X_c = X[y==c]
            self._mean[idx, :] = X_c.mean(axis=0)
            self._var[idx, :] = X_c.var(axis=0)
            self._priors[idx] = X_c.shape[0]/float(n_samples)
        self._var[:,:] += self.eps # adding epsilon to prevent variance from causing divide by 0

    def predict(self, X):
        y_pred = [self._predict(x) for x in X]
        return np.array(y_pred)

    def _predict(self, x):
        posteriors = []

        # calculate posterior probability for each class
        for idx, c in enumerate(self._classes):
            prior = np.log(self._priors[idx])
            posterior = np.sum(np.log(self._pdf(idx, x)))
            posterior = prior + posterior
            posteriors.append(posterior)
            
        # return class with highest posterior probability
        return self._classes[np.argmax(posteriors)]
            

    def _pdf(self, class_idx, x):
        mean = self._mean[class_idx]
        var = self._var[class_idx]
        numerator = np.exp((x - mean)**2 / -(2 * var))
        denominator = np.sqrt(2 * np.pi * var)
        return numerator # removing denominator (our goal is only to get the class not the probabilities)
